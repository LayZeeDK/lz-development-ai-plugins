Structure:
1. **When to use**: Integration-testing components that use browser APIs (LanguageModel, WebGPU, WebNN, Web Audio, Canvas, etc.) -- these APIs are unavailable in jsdom/happy-dom. Also for testing visual rendering, CSS, and real DOM behavior.

2. **Vitest Projects Config** (unit + browser split):
   - Show the complete `vite.config.ts` (or `vitest.config.ts`) with `test.projects` array
   - Unit project: `include: ['tests/unit/**/*.{test,spec}.ts']`, `environment: 'node'`
   - Browser project: `include: ['tests/browser/**/*.{test,spec}.ts']`, `browser.enabled: true`
   - Use the factory function import: `import { playwright } from '@vitest/browser-playwright'`
   - NOT the deprecated string format `provider: 'playwright'`

3. **Branded browser channels**:
   - `channel: 'chrome'` for Chrome (LanguageModel/Gemini Nano), `channel: 'msedge'` for Edge (Phi-4-mini)

   - Fallback: bundled `chromium` if branded channel unavailable
   - Example error and how to detect/handle it

4. **Browser instances configuration**:
   - `browser.instances: [{ browser: 'chromium' }]` (NOT deprecated `browser.name`)
   - Instance-level overrides are full replacements, not merges (document this pitfall prominently)
   - `actionTimeout` on the provider for test stability

5. **Framework-specific render packages**:
   - `vitest-browser-react` for React components
   - `vitest-browser-vue` for Vue components
   - `vitest-browser-svelte` for Svelte components
   - `vitest-browser-angular` for Angular components
   - Show import pattern: `import { render } from 'vitest-browser-react'`
   - Show assertion pattern: `await expect.element(screen.getByRole(...)).toHaveTextContent(...)`

6. **Agent reporter** for token-efficient output:
   - `vitest run --reporter=agent` or auto-detected via `AI_AGENT=1` env var
   - Only prints failing tests and final summary
   - Significantly reduces token usage when running inside Claude Code

7. **Isolation behavior**:
   - File-level isolation (one page per test file), NOT per-test
   - Group tests that share state into the same file
   - Clean up between tests within a file

8. **Key pitfalls** (from RESEARCH.md):
   - Branded channel not installed -> fallback to bundled chromium
   - Instance-level overrides replace parent options entirely
   - jsdom/happy-dom cannot test browser AI APIs -> must use Browser Mode

Source all configuration from RESEARCH.md Vitest 4.1 patterns. Use the exact API from Vitest 4.x (factory function provider, instances array, projects config).

**Part A: Create `plugins/application-dev/skills/vite-plus/SKILL.md`** (~200-300 lines) as a self-contained skill document.

Structure:
1. **When to use**: Greenfield web projects when compatible with the chosen tech stack. Preference, not mandate. Falls back to plain Vite when incompatible.

2. **Installation**:
   - macOS/Linux: `curl -fsSL https://vite.plus | bash`
   - Windows: `irm https://vite.plus/ps1 | iex`
   - Note: vp is a standalone CLI, not an npm package

3. **vp CLI commands** with descriptions and flags:
   - `vp create` -- scaffold new project (interactive or with template name)
   - `vp create <template>` -- templates: `vite`, `react-router`, `vue`, `svelte`
   - `vp dev` -- start dev server (Vite 8)
   - `vp check` -- format (Oxfmt) + lint (Oxlint) + typecheck (tsgo) in single pass
   - `vp check --fix` -- auto-fix lint and format issues
   - `vp test` -- run tests (Vitest 4.1)
   - `vp build` -- production build (Rolldown)
   - `vp run <task>` -- run custom tasks defined in vite.config.ts
   - `vp migrate` -- migrate existing Vite project to Vite+

4. **Unified vite.config.ts** -- single config for entire toolchain:
   - Show complete example with plugins, lint options, test config, and run tasks
   - `lint.options.typeAware: true` for type-aware lint rules
   - `lint.options.typeCheck: true` for full tsgo type checking in `vp check`
   - `test` section for Vitest configuration
   - `run.tasks` for custom task definitions

5. **Framework support and compatibility**:
   - Fully supported: React, Vue, Svelte, Solid, react-router
   - Partially supported/experimental: TanStack Start, Nuxt (incomplete integration)
   - Not supported: Angular (tsgo does not support Angular compiler plugins)
   - When Vite+ is incompatible, fall back to plain Vite + ESLint + Prettier + tsc

6. **Bundled tool versions** (as of alpha, March 2026):
   - Vite 8, Vitest 4.1, Oxlint 1.52, Oxfmt beta, tsgo (TypeScript Go port)
   - Rolldown for production builds

7. **Known limitations**:
   - Alpha software -- expect rough edges
   - Oxlint cannot lint Vue/Svelte templates (only script parts)
   - tsgo is experimental -- does not support Angular decorator metadata or custom compiler transformations
   - `vp check` with tsgo may fail on Angular projects -> fall back to `npx tsc --noEmit`

8. **CI integration with vp**:
   - `vp check` replaces separate lint + format + typecheck commands
   - `vp test` runs Vitest
   - `vp build` does production builds
   - The Generator's CI diagnostic becomes simpler when Vite+ is used

Source all patterns from RESEARCH.md Vite+ section. Use official viteplus.dev and VoidZero announcement as the authority.
