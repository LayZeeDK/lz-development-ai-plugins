    - Frontend SPA / interactive UI: Testing Trophy -- integration tests (Vitest Browser or Testing Library) as bulk, e2e for critical user flows, unit tests for pure utilities only

    - CLI / utility tool: Pyramid -- unit tests for logic/parsing, e2e = command invocation
    - Data / algorithmic app: Pyramid -- unit tests for algorithms and transforms
  - If the app uses browser AI APIs (LanguageModel, WebGPU, WebNN), set up Vitest Browser Mode: Read `${CLAUDE_PLUGIN_ROOT}/skills/vitest-browser/SKILL.md` for projects config with branded channels
  - E2e framework: install Playwright Test if the app has 3+ pages or complex user flows
- One coherent "development environment" commit -- the Generator does NOT start writing features without quality feedback
- Create or extend .gitignore (same as current Step 2)
- First git commit after setup
