<feature>
  <name>CLI Crash Recovery and Static Serve Foundation</name>
  <files>plugins/application-dev/scripts/appdev-cli.mjs, plugins/application-dev/scripts/test-appdev-cli.mjs</files>
  <behavior>
    resume-check subcommand:
    - Returns { next_action: "plan" } when step is "plan" and SPEC.md missing
    - Returns { next_action: "generate" } when step is "plan" and SPEC.md exists with ## Features
    - Returns { next_action: "generate" } when step is "generate" and build output missing (state.build_dir unset or dir missing)
    - Returns { next_action: "evaluate" } when step is "generate" and build output exists
    - Returns { next_action: "spawn-both-critics" } when step is "evaluate" and no valid summaries
    - Returns { next_action: "spawn-perceptual-critic" } when projection summary valid but perceptual missing
    - Returns { next_action: "spawn-projection-critic" } when perceptual summary valid but projection missing
    - Returns { next_action: "compile-evaluation" } when both summaries valid but EVALUATION.md missing
    - Returns { next_action: "round-complete" } when EVALUATION.md valid but git tag missing
    - Returns { next_action: "generate", round: N+1 } when current round fully complete
    - Reads expected critics from state.critics (default ["perceptual", "projection"])
    - Deletes corrupt critic directory when summary.json is invalid (JSON parse fail or missing required fields)
    - Validates summary.json: exists + JSON.parse succeeds + critic/dimension/score fields present
    - Validates EVALUATION.md: exists + contains "## Scores"
    - Checks git tag via git tag -l "appdev/round-N"
    - Returns { next_action: "summary" } when step is "summary"

    static-serve subcommand:
    - Starts a detached serve process for --dir with --port (default 5173)
    - Records { dir, pid, port, spa } in state.servers[]
    - Uses --spa flag to pass -s (single-page) to serve
    - Is idempotent: if server already running for same dir, returns existing entry
    - Validates existing PIDs are alive before claiming server is running (stale PID detection)
    - Uses mutex (mkdirSync) for concurrent start safety (same pattern as install-dep)
    - Finds available port starting from 5173, incrementing on conflict
    - --stop flag kills all tracked servers and clears state.servers[]
    - Uses taskkill /PID /T /F on Windows, process.kill(-pid, SIGTERM) on Unix
    - Health check: waits up to 5 seconds for port to respond after spawn

    update extensions:
    - --build-dir <path> sets state.build_dir
    - --spa true|false sets state.spa (parsed as string, compared === "true")
    - --critics <list> sets state.critics as comma-split array
    - --step is no longer required when only setting build-dir, spa, or critics

    delete/complete cleanup:
    - cmdDelete stops all servers (kills PIDs) before deleting state file
    - cmdComplete stops all servers before writing final state

  </behavior>
