<feature>
  <name>check-assets subcommand for appdev-cli.mjs</name>
  <files>plugins/application-dev/scripts/appdev-cli.mjs, tests/appdev-cli-check-assets.test.mjs</files>
  <behavior>
    **Input parsing:**
    - `node appdev-cli.mjs check-assets` -- uses default path `ASSETS.md` in cwd
    - `node appdev-cli.mjs check-assets --file path/to/ASSETS.md` -- custom path
    - Missing file -> error JSON on stdout, exit code 1

    **ASSETS.md parsing:**
    - Parses Markdown table rows for URLs in the "URL" column
    - Skips rows where URL is "local", "N/A", "n/a", or empty
    - Extracts all http/https URLs

    **URL verification (for each external URL):**
    - HEAD request first (fast, no body download)
    - If HEAD returns 403 or 405, retry with GET + User-Agent header
    - 5-second timeout via AbortController
    - Check content-type for image URLs: if URL ends in image extension but content-type is not image/*, flag as soft-404
    - Return per-URL result: { url, status, ok, contentType, soft404, error }

    **Output:**
    - Success (all URLs OK): `{ total, checked, passed, failed, warnings, results: [...] }` exit code 0
    - Some failures: same structure, exit code 0 (warnings, not errors -- per CONTEXT.md "diagnostic, not gate")
    - Parse error (no ASSETS.md, malformed): error JSON on stdout, exit code 1

    **Test cases:**
    - Test 1: Missing ASSETS.md file -> error JSON + exit 1
    - Test 2: ASSETS.md with only "local" URLs -> 0 checked, all pass
    - Test 3: ASSETS.md with valid http URLs -> correct status extraction (mock fetch)
    - Test 4: Soft-404 detection -> image URL returning text/html flagged
    - Test 5: HEAD-to-GET fallback -> 403 on HEAD triggers GET retry (mock fetch)
    - Test 6: Timeout handling -> AbortError mapped to timeout error
    - Test 7: No --file flag -> defaults to ASSETS.md in cwd
  </behavior>
  <implementation>
    Add `cmdCheckAssets(argv)` function to appdev-cli.mjs following the existing subcommand pattern.

    **ASSETS.md parsing function** `parseAssetsTable(content)`:
    - Split content by newlines, find table rows (lines starting with `|`)
    - Skip header and separator rows
    - Find the URL column index by checking header row for "URL" (case-insensitive)
    - Extract cells by splitting on `|`
    - Return array of { asset, url } objects for rows with external URLs

    **URL check function** `checkUrl(url, timeout = 5000)`:
    - Use Node.js built-in `fetch` (available in Node 18+, no dependencies)
    - HEAD first, GET fallback on 403/405
    - AbortController for timeout
    - Content-type soft-404 detection for image extensions
    - Return structured result object

    **Main function** `cmdCheckAssets(argv)`:
    - Parse --file arg (default: join(process.cwd(), 'ASSETS.md'))
    - Read and parse the file
    - Run checkUrl for each external URL (sequential to avoid rate limiting)
    - Compute summary stats
    - Output structured JSON

    Add `"check-assets"` case to the switch statement and update the default error message.

    **Test file:** `tests/appdev-cli-check-assets.test.mjs` using Node.js built-in test runner (`node:test`). Mock `global.fetch` for URL verification tests. Use temp directories with synthetic ASSETS.md fixtures.
  </implementation>
