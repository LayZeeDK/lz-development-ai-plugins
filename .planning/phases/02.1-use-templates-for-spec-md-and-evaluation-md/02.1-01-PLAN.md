<objective>
Extract inline document format specifications from planner.md and evaluator.md into standalone template files in the references/ directory, then update both agent definitions to reference the templates instead of embedding the format.

Purpose: The EVALUATION.md format is a contract between the Evaluator agent and appdev-cli.mjs (regex-parsed). When the format is defined inline in agent instructions, it drifts -- agents paraphrase, reorder, or omit sections. A canonical template file becomes the single source of truth for both "what the agent produces" and "what the CLI expects to parse." The same principle applies to SPEC.md structure for the Planner.

Output: Two template files (SPEC-TEMPLATE.md, EVALUATION-TEMPLATE.md) in references/, two updated agent definitions (planner.md, evaluator.md) with template references replacing inline format blocks.
</objective>
