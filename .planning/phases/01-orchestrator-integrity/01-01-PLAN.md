<task type="auto">
  <name>Task 1: Create appdev-state.cjs CLI script</name>
  <files>plugins/application-dev/scripts/appdev-state.cjs</files>
  <action>
Create `plugins/application-dev/scripts/` directory and write `appdev-state.cjs`.

The script is a Node.js CJS module (no dependencies) that reads/writes `.appdev-state.json` in the current working directory. All output is JSON to stdout. Errors go to stderr with process.exit(1).

Subcommands (parsed from process.argv):

1. `init --prompt "<text>"` -- Creates `.appdev-state.json` with initial state:
   ```json
   {
     "prompt": "<text>",
     "step": "plan",
     "round": 0,
     "status": "in_progress",
     "exit_condition": null,
     "rounds": []
   }
   ```
   Errors if state file already exists (prevents accidental overwrite).

2. `get` -- Reads and outputs the current state file as JSON. Errors if file does not exist.

3. `update --step <step> [--round <N>] [--status <status>]` -- Updates specified fields. Valid steps: "plan", "generate", "evaluate", "summary", "complete". Valid statuses: "in_progress", "error", "complete". Round is an integer >= 0.

4. `round-complete --round <N> --verdict <PASS|FAIL> [--scores '<json>'] [--feature-count <N>]` -- Appends or updates the entry in the `rounds` array for the given round number. Scores JSON shape: `{"depth":N,"func":N,"design":N,"code":N,"total":N}`. Feature count is optional (null if omitted).

5. `complete --exit-condition <condition>` -- Sets status to "complete" and exit_condition. Valid conditions: "PASS", "PLATEAU", "REGRESSION", "SAFETY_CAP".

6. `delete` -- Deletes the state file. No error if file does not exist.

7. `exists` -- Outputs `{"exists": true/false}` indicating whether state file is present. Never errors.

Argument parsing: use a simple argv loop, no external libraries. `--prompt` value can contain spaces (consume until next `--` flag or end of args). `--scores` value is a single JSON string argument.

Validation: reject unknown subcommands, missing required args, invalid enum values. On validation error, print JSON `{"error": "<message>"}` to stderr and exit 1.

The state file path is always `path.join(process.cwd(), '.appdev-state.json')`.
  </action>
