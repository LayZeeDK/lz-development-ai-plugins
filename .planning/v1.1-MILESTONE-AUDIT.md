## Integration Findings

### CRITICAL: install-dep --dev vs --package Flag Mismatch

**Agents call:** `node *appdev-cli* install-dep --dev sharp imghash leven`
**CLI expects:** `--package <name>` as required argument
**Result:** parseArgs never sets args.package; cmdInstallDep exits with `{"error":"Missing required argument: --package <name>"}`
**Impact:** Perceptual-critic cannot install sharp/imghash/leven for asset quality validation. Projection-critic cannot install ajv for schema validation. Both critics will get JSON error responses.
**Affects:** ENSEMBLE-04

### MEDIUM: SAFETY_CAP Path Missing static-serve --stop

The SAFETY_CAP wrap-up path spawns a wrap-up Generator without first stopping the old static-serve. If build dir is unchanged, the idempotent check reuses the stale server. New build content is not served.
**Affects:** RECOVERY-03

### MEDIUM: @playwright/test Not Guaranteed Installed

SKILL.md Step 0.5 installs `@playwright/cli` but not `@playwright/test`. Generator installs it conditionally (3+ pages or complex flows). For simple single-page apps, `npx playwright test` will fail.
**Affects:** PLAYWRIGHT-02, PLAYWRIGHT-04

### MEDIUM: baseURL Configuration Gap

Projection-critic Step 0 says to set baseURL but doesn't specify how. PLAYWRIGHT-EVALUATION.md skeleton test lacks `test.use({baseURL})`. If Generator's playwright.config.ts points to dev server port while static-serve uses a different port, all acceptance tests hit the wrong host.
**Affects:** PLAYWRIGHT-02, PLAYWRIGHT-04
