<decisions>
## Implementation Decisions

### Testimonial Content Policy
- Generator must follow SPEC-grounded testimonials: if SPEC includes testimonials sourced from the real website, use those
- If no real testimonials available, use clearly-marked placeholders (e.g., "[Placeholder - Client Name]") with generic roles
- Never name real people or real customer companies in fabricated testimonials
- This extends the entity research protocol added in 260405-0az from Planner to Generator

### Browser Channel Enforcement
- Default to `msedge` channel (Phi-4 Mini) for all Playwright browser launches
- Fallback chain: msedge -> chrome -> bundled chromium
- Enforce in both Generator (e2e tests, browser-built-in-ai) and all three Critics (Playwright evaluation)
- Rationale: Edge ships Phi-4 Mini which is more capable than Chrome's Gemini Nano for LanguageModel API

### Convergence Logic (PASS Exit Condition)
- Change `determineExit()` in appdev-cli.mjs: PASS requires ALL dimensions scoring 10 (perfect scores)
- If all dimensions meet thresholds but are not all 10, fall through to EMA escalation logic
- PLATEAU becomes the normal "good enough" exit -- scores meet threshold AND convergence detected
- Existing REGRESSION, SAFETY_CAP exits unchanged
- Original design (PASS on threshold-met) was from Phase 2. Consensus.dk test proved it allows premature exit when critics are lenient on round 1.

### Critic Scoring Calibration (Three Structural Mitigations)
Grounded in GAN/WGAN, Cybernetics, and Turing test theory:

1. **Round 1 Score Prior + Embedded Anchors** (Ashby's Requisite Variety + Dennett's Interrogator Effort)
   - Add to each critic: "First-generation applications typically score 3-5. Round 1 score above 6 requires explicit comparison against calibration scenario."
   - Embed the single most relevant "below threshold" calibration scenario directly in each critic's agent .md file
   - Restores full 1-10 range usage (Ashby: information capacity) and forces justification (Dennett: strong interrogation)

2. **Mandatory Defect Quota** (Beer's Algedonic Signal + Dennett's Interrogator Effort)
   - Add to each critic: "Minimum 3 findings before any score. Round 1 apps always have issues."
   - Forces pain channel open (Beer: prevents somnolent state) and mandates effort (Dennett: detection accuracy = f(effort))

3. **One-Sided Label Smoothing -- Score Cap** (Salimans et al. 2016 + Goodfellow 2014)
   - Round 1: max achievable score is 8 per dimension
   - Round 2+: max achievable score is 9 per dimension
   - Perfect 10 is never achievable
   - Prevents discriminator overconfidence (Goodfellow: vanishing gradients) via one-sided smoothing (Salimans: soften ceiling, keep floor honest)

### Claude's Discretion
- Specific wording and placement of calibration anchors within each critic agent .md
- Exact format of defect quota enforcement language
- Whether score cap is enforced in critic prompts or in the CLI aggregator (prefer CLI for structural enforcement)
