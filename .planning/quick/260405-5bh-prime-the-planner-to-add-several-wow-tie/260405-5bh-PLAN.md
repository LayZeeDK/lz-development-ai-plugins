  <action>
Create a new reference doc at `plugins/application-dev/skills/application-dev/references/ai-feature-inspiration.md`.
This is a principles-first teaching document (~80-120 lines, ~1,500 tokens), NOT a feature catalog.
It sits alongside `frontend-design-principles.md` (68 lines) and `acceptance-criteria-guide.md` (70 lines).

Structure with these sections:

1. **The Wow Spectrum** (~25 lines): Define Regular vs Wow tier with 3-4 contrastive pairs.
   - Regular: chatbot bolted onto sidebar, cloud API wrapper, generic "ask AI" button.
   - Wow: on-device inference via WebGPU, multi-modal pipelines chaining capabilities,
     context-aware intelligence that understands the product domain.
   Use concrete one-line examples (not full paragraphs) to show the contrast.

2. **Thinking Heuristics** (~20 lines): 6-8 principles that teach the Planner HOW to think
   at the Wow tier. Each heuristic is 1-2 lines. Key heuristics to encode:
   - On-device over cloud API (WebGPU/WASM signals ambition, privacy as differentiator)
   - Multi-modal pipelines (chaining voice -> text -> action -> visualization)
   - Context-aware intelligence (AI that understands the product domain, not generic wrapper)
   - Real-time over batch (live inference during interaction, not upload-then-wait)
   - Depth over breadth (one deeply integrated AI feature outshines five shallow ones)
   - Emergent behavior (AI that learns, adapts, or surprises)
   - Privacy as a feature ("all processing happens locally" is itself a selling point)

3. **AI Capability Menu** (~30 lines): A lightweight one-liner-per-category menu that
   triggers the Planner's imagination. NOT detailed examples. Categories drawn from the
   taxonomy: generative (text, image, audio, video, 3D, code, SVG, procedural content),
   analytical (classification, detection, recognition, forecasting, knowledge graphs),
   transformation (OCR, speech, translation, enhancement, style transfer, format conversion),
   interactive (autocomplete, recommendations, avatars, game AI, tutoring, semantic search),
   accessibility (alt-text, captioning, simplification, sign language), agentic (autonomous
   agents, workflow orchestration). One line each -- just enough to expand the vocabulary.

4. **Browser AI Capabilities** (~15 lines): What is actually possible in a browser today.
   Lightweight inventory -- the Generator has implementation skills, so this is "what's
   possible" not "how to do it." Include: WebGPU for model inference, Transformers.js
   (Hugging Face models in-browser), ONNX Runtime Web, TensorFlow.js, MediaPipe (vision),
   Chrome Built-in AI APIs (Prompt, Summarizer, Rewriter, Translator), Web Audio API,
   WebAssembly for compute-intensive tasks, Web Workers for parallel processing, IndexedDB
   for local model/data persistence.

5. **Anti-Patterns** (~15 lines): Regular-tier traps to avoid. Flag these explicitly:
   - A chatbot bolted onto the sidebar with no domain knowledge
   - "AI-powered" labels on features that are just API calls with no product integration
   - Generic "ask AI to generate X" without contextual awareness
   - Same AI features regardless of product domain (every app gets a chatbot)
   - AI as an add-on section rather than woven into the product's core workflows
   - Overloading with many shallow AI features instead of a few deep ones

6. **Integration Depth** (~15 lines): How to weave AI INTO the product rather than bolting
   it on. Key principles:
   - The best AI features amplify what the product already does
   - A music app's AI should understand music; a game maker's AI should understand game design
   - AI should appear where the user is already working, not in a separate panel
   - Chain AI capabilities into pipelines (e.g., voice -> transcription -> action)
   - Consider: what would be IMPOSSIBLE without AI? Those features are the most impressive

Keep the entire document under 120 lines. Use short, punchy prose -- not academic paragraphs.
The Planner already knows about these technologies from training; it needs ambition calibration
and heuristics, not a textbook.

Add a note at the top: escape hatch for prompts that explicitly exclude AI features or
describe purely static content sites.
